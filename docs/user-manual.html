产品简介

Metastellar PPT 是一款基于 Cursor AI Agent 的演示文稿生成服务。它专为营销机构向客户呈现方案而设计，默认产出高档、留白充足、层级清晰的提案级 PPT。

🎯 定位 Agency → Client 提案、战役方案、Retainer 汇报
🎨 设计系统 Metastellar 蓝色 Tailwind 色板 + 7 种 agency 高档版式
📦 输出 可下载 .pptx + 每页 PNG 预览 + 可复现 build.py
🌐 语言 界面与幻灯片均支持中文 / English / 日本語
无需 Access Token，打开页面即可使用（受 IP 速率限制保护）。

快速开始

本地运行（开发者）

export CURSOR_API_KEY=your_key
pip install -r requirements.txt
playwright install chromium
uvicorn app.main:app --reload --port 8080

浏览器访问 http://localhost:8080

网页端生成（普通用户）

  1. 撰写需求 — 在左侧文本框描述客户、战役目标、页数与要点；可上传 brief、数据文档。
  2. 选择参数 — 设置页数、信息密度（演讲型/阅读型）、幻灯片语言。
  3. 生成 PPT — 点击按钮，右侧工作台实时显示 Agent 进度。
  4. 预览与下载 — 完成后查看每页 PNG 预览，下载 deck.pptx。
  5. 在线编辑（可选） — 在「在线编辑」框输入修改指令，Agent 会更新并重新渲染。

界面使用

左侧面板 — 撰写需求

  • 需求文本框：核心输入。写清客户是谁、要什么方案、关键数据与约束。
  • 试试：点击示例 chip 可快速填充模板 brief。
  • 页数：1–6 页（可在服务端通过 MAX_PAGES 调整上限）。
  • 信息密度：演讲型 / 阅读型（见下文）。
  • 幻灯片语言：生成内容的语言，与界面语言独立。
  • 参考文档：支持 PDF、Word、PPT、Excel、图片、Markdown、纯文本。
  • 对话模式：开启后先出大纲，确认后再生成（Beta）。

右侧面板 — 工作台

  • 待命：展示 Agent 四步工作流程说明。
  • 运行中：流式显示思考、工具调用、状态与文本输出。
  • 完成：幻灯片预览图 + 下载链接。
  • 生成过程中可点击停止中断当前任务。

界面右上角可切换界面语言（中文 / EN / 日本語）。

生成模式

模式说明适用场景
一键生成（默认） 单次提交，Agent 自动完成理解 → 设计 → 构建 → 自检，无需等待确认。 Brief 清晰、赶时间、内部分享
对话模式（Beta） 首轮只出逐页大纲 + 追问；用户回复「确认」/ OK 后才开始生成 .pptx。 复杂提案、需要先对齐结构
在线编辑 生成完成后，用自然语言描述修改（如「标题改成…」「第三页换成 KPI 页」）。 小修小补、迭代优化
对话模式需要服务保持单实例（--max-instances 1），否则重启后会话上下文会丢失。

信息密度

演讲型（Speaker-led） 默认推荐。一页一观点，大标题、1–3 条要点、大留白。适合现场向客户提案。
阅读型（Reading-first） 信息更自洽，允许多列卡片、表格与 4–6 条要点。适合邮件发送或异步审阅。

内容过多时，Agent 会拆页而非缩小字号。若 brief 信息量大，请适当增加页数或选阅读型。

撰写有效 Brief

好的 brief = 好的提案。建议包含以下要素：

要素示例
客户 / 品牌「某消费品牌华东区」
方案类型「Q3 社媒增长提案」「新品上市战役」
页数「8 页，含封面与结案」
核心信息「策略三支柱 + 时间线 + KPI 预期」
数据 / 素材上传 brief PDF、过往数据、竞品截图
受众「客户市场总监，现场汇报」→ 选演讲型

示例 brief

为某消费品牌做 8 页 Q3 社媒增长方案提案：
封面、挑战洞察、策略三支柱、战役时间线、KPI 预期、结案页。
演讲型，突出 +38% 触达与 2.4× ROAS 目标。
附上上传的 Q2 复盘 PDF。

提案叙事结构

Agent 默认按营销机构客户提案的经典叙事弧线组织内容：

顺序幻灯片角色作用
1封面客户名 + 方案名，第一印象
2议程今天讲什么
3挑战 / 背景客户现状与痛点
4核心洞察North Star 一句话
5策略支柱3–4 个可执行方向
6创意 / 渠道怎么做
7时间线战役节奏与里程碑
8KPI 预期大数字证明价值
9结案Thank you / Q&A

页数较少时自动压缩为：封面 → 大创意 → 支柱或 KPI → 结案。

版式适用时刻

设计规范：每页 ≤3 色、Segoe UI 字体、蓝/靛蓝主色 + 灰阶中性色。

在线编辑

首次生成完成后，「在线编辑」区域会出现。用自然语言描述修改即可，例如：

  • 「把封面客户名改成 ABC Brand」
  • 「第三页换成 KPI 页，三个指标：触达 +40%、ROAS 2.5×、CPA -20%」
  • 「整体色调再偏深蓝一点」
  • 「增加一页竞品分析」

Agent 会更新 build.py、重新导出 .pptx 并刷新预览图。未提及的页面尽量保持不变。

输出文件

每次生成会话产出：

文件说明
output/deck.pptx最终 PowerPoint，16:9 宽屏
output/preview-*.png每页渲染预览，用于自检溢出与配色
build.py可复现的构建脚本，编辑后重新运行即可
output/deck_meta.json标题与下载文件名元数据

下载文件名格式示例：品牌增长提案-2026-07-06.pptx

部署说明

推荐部署到 Google Cloud Run：

gcloud run deploy metastellar-ppt \
  --source . \
  --project YOUR_PROJECT \
  --region asia-east1 \
  --allow-unauthenticated \
  --memory 2Gi --cpu 2 --timeout 600 \
  --concurrency 4 --max-instances 1 \
  --set-secrets CURSOR_API_KEY=CURSOR_API_KEY:latest \
  --set-env-vars CURSOR_MODEL=composer-2.5
  • 对话模式建议 --max-instances 1 保持 Agent 会话。
  • Docker 镜像已包含 LibreOffice（预览渲染）与 Playwright（URL 截图）。
  • 可选 GCS_BUCKET 持久化会话，应对实例重启。

常见问题

生成失败或超时？

尝试减少页数、简化 brief，或拆成多次生成后合并。检查服务端 CURSOR_API_KEY 是否有效。

预览图文字溢出？

Agent 有自检环节，但偶发仍可能溢出。使用「在线编辑」要求精简文案或拆页。

能用自己的 Logo 吗？

在 brief 中说明并提供 Logo 图片上传；Agent 会尝试放入封面等位置。

支持动效吗？

当前输出标准 .pptx 静态幻灯片。HTML 动效版为后续规划方向。

数据安全？

请勿上传涉密资料。上传文件仅用于当次生成，会话默认 24 小时后自动清理。
